🎁 お詫び・プレゼント配布システム 運用マニュアル

ビルド（アプリアップデート）を通さずに、GitHub上の announcements.json を更新するだけで、ユーザーにAIクレジットを配布する方法を解説します。

1. 配信の基本ルール

  • アプリは起動時にJSONを読み込み、「未読のお知らせ」を上から順にスキャンします。
  • giftCredits が見つかると、自動的にクレジットを加算します。
  • 一度加算されたお知らせは、既読にする前でもDBに記録されるため、再起動しても二重配布されません。
⚠️ 最重要：最新のお知らせは必ず一番上に追加してください
アプリは上から順に読み、前回保存した最新のIDに到達した時点でスキャンを終了します。古い情報の ID を書き換えても反応しません。

2. JSONの記述例

パターンA：全員に配布（お詫びなど）

[
  {
    "id": "20260320_fix_apology",
    "title": "同期不具合のお詫び",
    "body": "不具合のお詫びとして、AIクレジットを5つ付与しました。",
    "date": "2026/03/20",
    "category": "important",
    "giftCredits": 5
  }
]

パターンB：特定ユーザーのみに配布（個別補填など）

スライドメニューに表示される8桁のIDを指定します。複数のユーザーを指定することも可能です。

[
  {
    "id": "20260320_compensation_01",
    "title": "個別補填のお知らせ",
    "body": "お問い合わせの件について、補填のクレジットを付与しました。",
    "date": "2026/03/20",
    "category": "info",
    "giftCredits": 10,
    "targetIds": ["A1B2C3D4", "E5F6G7H8"]
  }
]

3. 付与の制限事項

合計10クレジットの上限チェック
アプリ内の最大保持数は10回です。現在のクレジット + プレゼント数 が10を超える場合、付与は保留されます。ユーザーがクレジットを消費して空きができた後、次回のアプリ起動時に自動で付与が再試行されます。

4. JSON全項目リファレンス

announcements.json で使用可能なすべての項目と説明です。

項目名 型 説明
id String 必須。 お知らせを識別する一意のID。日付+連番などを推奨。以前と同じIDを使うと配信されません。
title String 必須。 お知らせのタイトル。ポップアップやリストに表示されます。
body String 必須。 お知らせの本文。お詫びの内容などを記載します。
date String 必須。 表示用の日付（例: "2026/03/20"）。
category String 必須。 info (青), update (緑), important (赤) のいずれかを指定。
giftCredits Number 任意。付与するAIクレジット数。1〜10の数値を指定してください。
targetIds Array 任意。特定のユーザーのみに配る場合のIDリスト（例: ["ID1", "ID2"]）。省略または null で全員配布。
url String 任意。詳細な説明がある場合の外部リンクURL。
