🎁 お詫び・プレゼント配布システム 運用マニュアル

アプリのアップデート（ビルド）を行うことなく、GitHub上の announcements.json を書き換えるだけでユーザーにAIクレジットを配布する仕組みの解説です。

1. 配信の仕組み

  • アプリ起動時に指定のURLからJSONを取得します。
  • 前回のチェック以降に追加された「未読」のお知らせをすべてスキャンします。
  • お知らせに giftCredits が設定されている場合、自動的にユーザーのクレジットに加算します。

2. JSONの書き方

GitHubの announcements.json の リストの一番上 に新しい要素を追加してください。

全員に配布する場合

[
  {
    "id": "20260320_fix",
    "title": "同期不具合のお詫び",
    "body": "ご迷惑をおかけしました。お詫びとして5クレジット付与します。",
    "date": "2026/03/20",
    "category": "important",
    "giftCredits": 5
  }
]

特定のユーザーだけに配布する場合

お問い合わせをくれたユーザーのID（スライドメニューに表示）を指定します。

{
  "id": "20260320_support_A",
  "title": "個別補填のお知らせ",
  "body": "お問い合わせの件につきまして、10クレジットを付与しました。",
  "date": "2026/03/20",
  "category": "info",
  "giftCredits": 10,
  "targetIds": ["A1B2C3D4"]
}

3. 運用上の重要なルール

⚠️ 最新のお知らせは必ず一番上（インデックス0）に追加してください。
アプリは上から順に読み込み、既読のIDを見つけた時点でスキャンを終了します。
⚠️ ID（id）は必ずユニークな文字列にしてください。
以前と同じIDを使うと、アプリが「既に読んだ」と判断してプレゼントが配布されません。
🛡️ 二重配布の防止機能
お知らせを閉じる前にアプリを再起動しても、DB側に「処理済みID」を記録しているため、同じお知らせから二度クレジットが増えることはありません。
📊 合計10クレジットの制限
配布時に 現在の残数 + プレゼント数 が10を超える場合、付与は保留されます。ユーザーがクレジットを消費して空きができた後の次回起動時に、自動で付与が実行されます。

4. ユーザーIDの確認方法

特定ユーザーへの配布に必要なIDは、以下の場所でユーザーに確認してもらってください。

  • アプリトップ画面 ＞ 左上のメニュー（三本線）をタップ
  • スライドメニューの 一番下 に表示されている ID：XXXXXXXX
