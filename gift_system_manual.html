🎁 お詫び・プレゼント配布システム 運用マニュアル

ビルドを通さずに、GitHub上の announcements.json を更新するだけで、ユーザーにAIクレジットを配布する方法を解説します。

1. 配布の基本ルール

  • アプリは起動時にJSONを読み込み、「未読のお知らせ」を上から順にスキャンします。
  • giftCredits が設定されている場合、対象ユーザーへ自動的にクレジットを加算します。
  • 一度加算されたお知らせは、DBに記録されるため、再起動しても二重配布されません。
⚠️ 最重要：最新のお知らせは必ずリストの「一番上」に追加してください
アプリは上から順に読み、前回読んだ最新のIDに到達した時点でスキャンを終了します。下の方に新着を追加しても無視されます。

2. JSONの記述例

パターンA：全員に一斉配布

[
  {
    "id": "20260320_apology_all",
    "title": "同期不具合のお詫び",
    "body": "不具合のお詫びとしてAIクレジットを5つ付与しました。",
    "date": "2026/03/20",
    "category": "important",
    "giftCredits": 5
  }
]

パターンB：特定の複数ユーザーに配布

スライドメニューのIDを指定します。複数をカンマ区切りで指定可能です。

[
  {
    "id": "20260320_compensation_A_B",
    "title": "個別補填のお知らせ",
    "body": "お問い合わせの件について、10クレジットを付与しました。",
    "date": "2026/03/20",
    "category": "info",
    "giftCredits": 10,
    "targetIds": ["A1B2C3D4", "E5F6G7H8"]
  }
]

パターンC：お知らせを「追加」する場合（カンマ区切り）

すでにお知らせがある場合、一番上の { } の後ろにカンマ , を打って、新しい { } をその上に配置します。

[
  {
    "id": "20260321_new_info",
    "title": "新しいお知らせ",
    "body": "これが一番上に表示され、プレゼント対象になります。",
    "date": "2026/03/21",
    "category": "info",
    "giftCredits": 1
  },
  {
    "id": "20260320_apology_all",
    "title": "古いお知らせ",
    "body": "カンマで区切って下に追加されていきます。",
    "date": "2026/03/20",
    "category": "important"
  }
]

3. 制限事項と安全機能

🛡️ 二重配布の防止
クレジット付与は「JSONを読み込んだ瞬間」にDBへ LAST_GIFT_PROCESSED_ID として記録されます。そのため、ポップアップを閉じる前にアプリを落としても、二度受け取ることはありません。
📊 合計10クレジットの上限チェック
アプリ内の最大保持数は10回です。現在のクレジット + プレゼント数 が10を超える場合、付与は一旦「保留」されます。ユーザーがクレジットを消費して空きができた後、次回のアプリ起動時に自動で付与が再試行されます。

4. JSON全項目リファレンス

項目名 型 説明
id String 必須。 一意のID。以前と同じIDを使うと配信されません。
title String 必須。 お知らせのタイトル。
body String 必須。 お知らせの本文。
date String 必須。 表示用の日付（例: "2026/03/20"）。
category String 必須。 info, update, important のいずれか。
giftCredits Number 任意。付与するAIクレジット数。指定しない場合は 0。
targetIds Array 任意。IDリスト（例: ["ID1", "ID2"]）。省略で全員配布。
url String 任意。詳細ページがある場合のURL。
